# 项目：工单系统与飞书话题群双向同步集成方案

## 版本: v1.1 状态: 待评审 日期: 2026-01-30 修订记录: v1.1 增加状态标记与满意度评价流程

### 第一部分：产品需求文档 (PRD)

#### 项目背景与目标

当前产品缺乏一个聚合的工单反馈渠道。为了提升客服响应速度并实现研发/产品团队的信息同步，我们需要构建一套**“工单即话题”**的客服系统。

核心目标：

- 用户侧： 在产品内获得即时的客服对话体验，并在服务结束后对服务质量进行评价。发起一次对话，在话题群创建一个话题，由内部客服、运营、技术跟进，直到话题被close。
- 客服/内部侧： 利用飞书（Lark）的**“话题群”功能，将每一个工单聚合为一个独立的话题帖子，并能通过卡片颜色/标签直观识别工单状态**。
  - 通过飞书机器人的指令回复用户信息，例如输入 /c xxx ，xxx 将会发送给用户，其他信息为内部信息，不发送给用户

核心流程模型： 用户提交工单 -> 飞书话题群新增话题 -> 客服接单处理 -> 客服结单 -> 用户评价 -> 飞书卡片更新评分结果。
